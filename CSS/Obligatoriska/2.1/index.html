Externt länkad css-fil och dess fördelar

Istället för att skriva styling direkt i vårt HTML-dokument så kan vi skapa externa css-dokument som vi sedan länkar till i head-sektionen i vår HTML.
Detta gör det möjligt att ha vår css mer strukturerad då vi kan skapa olika css-dokument för olika delar av stylingen. Det gör det även möjligt att
på flera olika sidor länka till samma css-dokument vilket i sin tur gör att vi slipper skriva samma kod på flera ställen. Exempelvis om vi vill att alla
sidor ska ha samma tema (bakgrund, font, utseende på knappar etc etc).

Arv

Med arv betyder att styling görs på ett parent-element vilket gör att barnen till det element får samma styling.
Värt att notera är att alla typer av stylingproperties inte går att ärva. Man kan dock "tvinga" element att ärva från sin förälder genom nyckelordet "inherit".
Exempelvis så är padding ett property som inte ärvs automatiskt, så för att få ett barn att ärva detta kan vi göra följande: 


         p {
            padding: 20px;
         }

         #elementInParagraph {
            padding: inherit;
         }

Cascade

CSS står för Cascading Style Sheets och Cascade betyder processen av att kombinera CSS-filer och CSS-regler(styling) och lösa konflikter mellan dem.
Konflikterna uppstår om man stylar samma property av ett element på flera olika ställen. Dvs man kan styla olika properties av ett element på flera olika ställen och all
styling kommer appliceras, men om man exempelvis ändrar color på ett element på ett ställe och sedan ändrar font-color igen på samma element på ett annat ställe så uppstår en konflikt.
Vilken styling ska gälla när sidan presenteras?

För att lösa dessa typer av konflikter använder sig CSS av två regler:

Regel 1

Den första regeln är regeln som säger att om man stylar ett element med samma selektor så är det den senast applicerade stylingen den som "vinner".
Exempel: 


            h1 {
               color: blue;
            }

            h1 {
               color: red;
            }

I detta fall kommer våra h1 få färgen röd då de båda använder samma selektor och den styling som säger att elementet ska vara rött ligger efter den första som säger blå.

Regel 2

Den andra regeln är regeln om specifcitet. Den säger att stylingen som är gjord med den mest specifika selektorn "vinner".
Exempel: 


            #h1Tag {
               color: red;
            }

            h1 {
               color: blue;
            }

I ovan fall kommer vår h1 få färgen röd trots att vi har styling som säger att h1 ska vara blå efter. Detta pga av att en ID-selektor(#h1Tag) är mer specifik än type-selektorn(h1).

Selektorer

Tagg Attribut Klass ID Barn

Selektorer används i CSS för att välja vilket/vilka element man vill styla. För att styla alla h2 på denna sida skulle vi exempelvis kunna göra: 


         h2 {
            /*styling här*/
         }

I exemplet ovan är "h2" selektorn.

Tagg

En tagg-selektor är en selektor som man använder när man vill styla alla element av en viss typ. Exempel på det ser vi ovan när vi använder "h2" som selektor.

Attribut

En attribut-selektor är en selektor som används när man vill styla alla element som har samma attribut. Om ett/flera element har attributet "alt" kan vi nå det/dem såhär: 

            
               [alt] {
                  padding: 40px;
               }

Klass

Klass-selektorn används när man svill styla alla element med samma namn på klassattributet. Exempel: 

            
               .myParagraphs {
                  color: red;
               }

ID

ID-selektorn används när man vill ett styla element med ett visst namn på sitt ID-attribut. Exempel: 

            
               #myId {
                  color: red;
               }

Barn

Barn-selektorn används när man vill styla ett element som är barn till ett annat element. I nedan exempel vill vi styla alla p-element som är barn till header-element: 

            
               header > p {
                  color: red;
               }

Enheter

Enheter, units, är de olika sätten CSS använder sig av för att representera längd. Det finns absoluta typer och relativa typer:

Absoluta enheter

  • in - inches (1in = 96px = 2.54cm)
  • cm - centimeter
  • mm - millimeter
  • pt - points (1pt = 1/72 av 1in)
  • pc - picas (1pc = 12 pt)
  • px - pixels (1px = 1/96th av 1in)

Relativa enheter

  • em - Relativt till fontstorleken av elementet (2em betyder 2 gånger storleken av nuvarande font)
  • ex - Relativt till fontstorlek. 1ex är höjden för "x" av nuvarande font
  • % - Procentuell storlek relativ till storleken på föräldraelementet

Hur ett CSS-dokument är uppbyggt

Med CSS-dokument menar man den filen där man skriver sina CSS-regler(stylingen).

  • En CSS-regel består utav en selektor och en deklarationsdel. Deklarationsdelen omsluts av måsvingar.
  • Selektorn pekar på HTML-elementet man vill styla.
  • Deklarationsdelen består utav en eller flera styling-deklarationer separerade med semikolon.
  • Varje deklaration innehåller ett CSS-attribut och ett värde för det attributet.
  • Attributet och värdet separeras med semikolon.

Exempel:


         pre {
            margin: 0px;
            padding: 0px;
         }

         a {
            color: black;
         }

         a:visited {
            color: black;
         }

         .firstLevelSection {
            background-color: #f4f7ef;
            border: solid 1px;
            border-radius: 15px;
            border-color: #b3b3bf;
            margin: 20px;
         }
